# 과목 정리 프롬프트 (범용 템플릿)

`[과목명]`을 실제 과목으로 교체하여 새 과목 채팅 시작 시 붙여넣기.

---

## 붙여넣을 프롬프트

```
[과목명] 수업 정리를 도와줘. 업로드된 PDF는 강의 슬라이드이다.

■ 수업 설명 규칙:
1. 특정 챕터 정리를 요청하면 해당 PDF 자료를 ZIP으로 추출하여 슬라이드 전체를 읽고 정리를 시작한다.
   (cp [file].pdf [file].zip && unzip)
2. 채팅창에서 학습 가능하도록 충분히 상세하게 설명한다.
   - 각 개념의 동기(왜 필요한가)·직관·원리를 풀어서 설명
   - 슬라이드의 실제 코드·수식·표를 생략 없이 포함
   - 핵심 다이어그램/회로도를 ASCII로 재현
   - 개념 간 연결과 흐름을 자연스럽게 서술 (단순 나열 금지)
   - 중요한 주의사항(함정, 흔한 오해)을 명시적으로 짚어줌
3. 설명 단위: 챕터(또는 PDF) 1개씩. 설명 완료 후 "다음"이라고 하면 다음 자료로 진행.

■ 문서 작성 규칙 ("다음 반영" 요청 시 실행):
1. 파일 위치: GitHub(https://github.com/koyoume/public.git) system_expert 폴더
   - Knowledge: [과목명]_Knowledge.md / .pdf
   - 수업정리노트: [과목명]_수업정리노트.pdf
   - 세션 시작 시 git clone 또는 git fetch+reset으로 최신화

2. Knowledge 문서 구조 (오픈북 시험 최적화):
   ① 수식 모음 — 계산 문제용 표 형태
   ② 핵심 비교표 — 개념 비교 문제용 표 형태
   ③ 용어 정의 — 가나다순 표 형태
   ④ 주제별 상세 — 챕터별 핵심 bullet + ASCII 회로도/다이어그램 + 예시 문제 풀이
   ⑤ 시험 대비 체크리스트 — 계산/개념 문제 유형별

3. ④ 주제별 상세 작성 기준:
   - 단순 개념 요약 금지. 오픈북 시험에서 응용문제를 단계적으로 풀 수 있는 수준으로 작성
   - 관련 회로도/다이어그램을 ASCII로 함께 제시
   - 각 핵심 개념마다 예시 문제 + 단계별 풀이 포함
   - 수식은 변수 의미와 조건까지 명시

4. 수업정리노트 작성 기준:
   - 채팅창에서 설명한 수업 내용 전체를 별도 MD로 작성 후 PDF 변환
   - 각 챕터를 처음부터 풀어서 설명한 내용 그대로 (동기·직관·원리·다이어그램 포함)
   - 슬라이드의 실제 코드·수식이 빠짐없이 포함되어야 함

5. 파일 생성 (매 업데이트 시 아래 3종 모두):
   (A) Knowledge MD — present_files로 제공
   (B) Knowledge PDF — pandoc 변환 후 present_files 제공
     명령어: pandoc [md] -o [pdf] --pdf-engine=wkhtmltopdf -V geometry:margin=15mm -V papersize:a4
     PDF에서 제거할 항목:
       - 첫 줄 H1 제목
       - 업데이트 정보 라인 (> 삼성DS...)
       - 소제목 헤더 전체 (## ①~④ 줄 자체 삭제)
       - ⑤ 시험 대비 체크리스트 섹션 전체
   (C) 수업정리노트 PDF — 별도 MD 작성 후 PDF 변환
     명령어: pandoc [md] -o [pdf] --pdf-engine=wkhtmltopdf -V geometry:margin=20mm -V papersize:a4

6. GitHub push:
   - 위 3종 파일 모두 system_expert/ 폴더에 commit+push
   - git config: user.email "koyoume@users.noreply.github.com", user.name "koyoume"
   - remote: https://koyoume:[YOUR_GITHUB_TOKEN]@github.com/koyoume/public.git
   - 챕터 추가 시 기존 파일에 누적 업데이트 (덮어쓰기가 아닌 추가)
```
