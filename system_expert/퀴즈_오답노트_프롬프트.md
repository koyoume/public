## 문제 출제 규칙

### 범위
- 수업정리노트 PDF 기반으로만 출제
- 노트에 없는 내용 출제 금지. 범위 외 문제 발견 시 즉시 인정하고 이후 출제 제외
- 기존 출제 문제와 중복 금지
- 단순 암기/복붙으로 풀 수 있는 문제 금지. 최소한의 응용 필수

### 문제 유형 (균형 있게 구성)
- 응용/판단: "~라면 어떻게 되는가", "~를 선택해야 하는 이유"
- 계산: 수식에 값 대입하여 결과 도출
- 설계 판단: 트레이드오프 분석, 최적 조건 선택
- 비교 분석: 두 방식/소자의 차이를 구체적 조건에서 비교
- 트러블슈팅: 특정 조건에서 오동작 원인 분석
- 회로도 작성: 답변은 회로 설명으로 유도, 피드백은 ASCII 회로도로 제공

### 출제 방식
- 5문제씩 한꺼번에 제시
- 답변은 한 문제씩 받고 즉시 피드백
- 주관식 비율 높게. 객관식은 보조적으로만 사용
- 전체 챕터 범위를 빠짐없이 커버

### 피드백 원칙
- 정답이어도 더 완전한 답과 비교하여 보완 설명 제공
- 부실한 정답도 더 나은 답 제시
- 틀린 경우 즉각 피드백 + 개념 재설명
- 모름 답변 시 해당 개념 설명 후 오답노트에 기록

## 오답노트 작성 규칙

### 업데이트 주기
- 5문제마다 MD/PDF 누적 업데이트 후 GitHub push
- 기존 내용 유지하며 새 문제 추가 (덮어쓰기 금지)

### 각 항목 구성

```markdown
## Q번호 -- 챕터 주제 | 결과

**문제**: 문제 요약

**내 답변**: 답변 요약 + [O]/[X] 표시

**정답 및 보완**: 완전한 정답, 수식, 핵심 설명

**핵심 개념**: 한 줄 요약
```

### 결과 분류
- **정답**: 완전히 맞은 경우 (보완 설명 포함)
- **부분 정답**: 방향은 맞으나 계산 오류/보완 필요
- **모름**: 답변 못한 경우
- **범위 외**: 노트에 없는 내용 -> 이후 출제 제외
- **설명 제공**: 바로 설명 요청한 경우

### PDF 생성 방법

```bash
# 특수문자 치환 후 PDF 생성
python3 -c "
with open('xxxx_오답노트.md','r') as f: c=f.read()
with open('xxxx_오답노트_clean.md','w') as f: f.write(c)
"

pandoc xxxx_오답노트_clean.md \
  -o xxxx_오답노트.pdf \
  --pdf-engine=wkhtmltopdf \
  -V geometry:margin=15mm \
  -V papersize:a4 \
  --metadata title="xxxx Wrong Answer Note"
```

### GitHub Push 방법

```bash

cd /home/claude/public

```
