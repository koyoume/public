## Chapter 1: Computer Abstractions and Technology

### §1.1 컴퓨터 혁명과 배경

#### 왜 이 챕터를 배우는가?

컴퓨터 구조를 공부하기 전에, 먼저 "컴퓨터가 왜 이렇게 빠르게 발전했는가"와 "컴퓨터 시스템이 어떤 계층으로 구성되어 있는가"를 이해해야 한다. 이 챕터는 전체 과목의 로드맵에 해당한다. 성능을 어떻게 정의하고 측정하는지, 하드웨어와 소프트웨어가 어떻게 협력하는지에 대한 기본 프레임워크를 세운다.

#### 무어의 법칙 (Moore's Law)

컴퓨터 발전의 원동력은 **무어의 법칙**이다. "집적 회로의 트랜지스터 수가 약 2년마다 2배로 증가한다"는 경험적 관찰이다.

```
트랜지스터 수 (로그 스케일)
 50,000,000,000 ─ ●  (72-core Xeon, 2018~)
 10,000,000,000 ─
  1,000,000,000 ─ ● (Dual-core Itanium 2, 2006~)
    100,000,000 ─ ● (Pentium 4, 2000~)
     10,000,000 ─ ● (Pentium, 1993~)
      1,000,000 ─ ● (Intel 80486, 1989)
        100,000 ─ ● (Intel 80386, 1985)
         10,000 ─ ● (Intel 8080, 1974)
          1,000 ─ ● (Intel 4004, 1971)
               └──────────────────────────
                1970    1990    2010   2018
```

이 법칙 덕분에 과거에 불가능했던 응용들이 현실이 되었다: 자동차 컴퓨터, 스마트폰, 유전체 프로젝트, WWW, 검색 엔진, 그리고 기계학습/AI.

**주의사항:** 무어의 법칙은 물리 법칙이 아니라 **경험적 관찰**이다. 최근에는 미세공정의 물리적 한계로 인해 둔화되고 있으며, 이것이 병렬 처리(parallelism)가 중요해진 핵심 이유이다.

#### 컴퓨터의 분류

시대별로 분류가 조금씩 변화한다.

**2008년 분류:** Desktop / Server / Embedded  
**2013년 분류:** Personal Computer / Server / Supercomputer / Embedded

| 클래스 | 목적 | 핵심 설계 기준 |
|--------|------|---------------|
| **Personal Computer** | 범용, 다양한 소프트웨어 | 비용/성능 트레이드오프 |
| **Server** | 네트워크 기반 서비스 | 높은 용량, 성능, **신뢰성** |
| **Supercomputer** | 과학/공학 계산 | 최고 성능 (시장 비중은 작음) |
| **Embedded** | 시스템 내부 부품 | 전력/성능/비용 제약이 엄격 |

**함정 주의:** Embedded 컴퓨터는 눈에 보이지 않지만 **수량 기준으로 가장 많다.** 세탁기, 자동차 ECU, 공유기 등 모두 임베디드 컴퓨터이다.

#### PostPC 시대와 시장 변화

2011년경부터 스마트폰 출하량이 PC를 추월했다.

```
출하량 (백만 대)
1400 ─     ╲ Cell phone (피처폰)
1200 ─    ╱  ╲
1000 ─  ╱      ╲
 800 ─
 600 ─          ╱ Smart phone (급성장)
 400 ─    ──── PC (정체)
 200 ─  ╱  Tablet (급성장)
   0 ─
     2007  2009  2011  2012
```

두 가지 새로운 컴퓨팅 패러다임:

**PMD (Personal Mobile Device):** 배터리 구동, 인터넷 연결, 수백 달러 가격대. 스마트폰, 태블릿, 전자안경 등.

**Cloud Computing:** 창고 규모 컴퓨터(WSC)에서 SaaS 형태로 서비스. 소프트웨어의 일부는 PMD에서, 일부는 클라우드에서 실행. Amazon, Google이 대표적.

**핵심 통찰:** PostPC 시대에서는 "단일 기기의 절대 성능"보다 "전력 효율"과 "네트워크 연결성"이 설계의 핵심 제약 조건이 된다.

### §1.2 컴퓨터 구조의 8가지 위대한 아이디어 (Eight Great Ideas)

이 8가지 아이디어는 **이 과목 전체를 관통하는 설계 원칙**이다.

1. **Design for Moore's Law** — 설계 시점이 아니라 **완성 시점**의 기술 수준에 맞춰 설계. 칩 설계에 수 년이 걸리므로, 설계 시작 시점이 아니라 완성될 때의 트랜지스터 수를 기준으로 해야 한다.

2. **Use Abstraction to Simplify Design** — 복잡성을 계층적으로 숨겨서 각 수준에서 관리 가능하게. SW/HW 인터페이스(ISA)가 대표적 추상화 경계.

3. **Make the Common Case Fast** — 자주 발생하는 경우를 빠르게. 드문 경우를 최적화하는 것보다 훨씬 효과적. (→ Amdahl's Law와 직결)

4. **Performance via Parallelism** — 여러 작업을 동시에 수행하여 성능 향상.

5. **Performance via Pipelining** — 병렬 처리의 특수한 형태. 작업을 단계별로 나누어 동시에 여러 작업의 서로 다른 단계를 처리. (→ Ch4 파이프라인에서 핵심)

6. **Performance via Prediction** — 확실해질 때까지 기다리는 것보다, 예측하고 틀리면 복구하는 게 평균적으로 더 빠르다. (→ Branch prediction)

7. **Hierarchy of Memories** — 빠르고 비싼 메모리(SRAM)와 느리고 싼 메모리(DRAM, Disk)를 계층적으로 배치하여, 마치 크고 빠른 메모리가 있는 것처럼 보이게 한다. (→ Ch5 캐시)

8. **Dependability via Redundancy** — 개별 부품은 고장날 수 있으므로, 중복을 두어 신뢰성을 확보.

### §1.3 Below Your Program — 프로그램 아래에서 일어나는 일

#### 소프트웨어의 계층 구조

```
┌─────────────────────────────┐
│     Applications Software    │  ← 사용자가 작성 (HLL)
│  (브라우저, 게임, 오피스 등)    │
├─────────────────────────────┤
│      Systems Software        │
│  ┌─────────────────────────┐│
│  │ Compiler: HLL → 기계어   ││
│  │ OS: I/O, 메모리, 스케줄링 ││
│  └─────────────────────────┘│
├─────────────────────────────┤
│         Hardware             │
│  Processor, Memory, I/O     │
└─────────────────────────────┘
```

**Application Software:** 고수준 언어(C, Java, Python 등)로 작성.

**System Software:**
- **Compiler:** 고수준 언어를 기계어로 번역.
- **Operating System:** I/O 처리, 메모리/저장장치 관리, 태스크 스케줄링 및 자원 공유.

**Hardware:** 프로세서, 메모리, I/O 컨트롤러.

#### 프로그램 코드의 수준 (Levels of Program Code)

swap 함수의 변환 과정:

**High-level language (C):**
```c
swap(int v[], int k)
{int temp;
    temp = v[k];
    v[k] = v[k+1];
    v[k+1] = temp;
}
```

**↓ Compiler**

**Assembly language (RISC-V):**
```
swap:
    slli x6, x11, 3
    add  x6, x10, x6
    ld   x5, 0(x6)
    ld   x7, 8(x6)
    sd   x7, 0(x6)
    sd   x5, 8(x6)
    jalr x0, 0(x1)
```

**↓ Assembler**

**Binary machine language:**
```
00000000001011010011000010011
00000000011010010000001100011
00000000000000011001100010000011
00000000010000011001110000011
00000000011100001100000100011
00000000010100011010000100011
00000000000010000001100111
```

고수준 언어는 문제 영역에 가깝고 생산성과 이식성을 제공한다. 어셈블리 언어는 명령어의 텍스트 표현이며, 최종적으로 하드웨어가 실행하는 것은 0과 1로 인코딩된 **이진 기계어**이다.

**주의:** 이 과목에서 사용하는 ISA는 **RISC-V**이다. ARM이나 x86이 아닌 RISC-V를 사용하는 이유는 깔끔하고 규칙적인 설계 덕분에 교육용으로 적합하기 때문이다.

### §1.4 Under the Covers — 컴퓨터 하드웨어의 구성

#### 컴퓨터의 5대 구성 요소

데스크톱이든 서버든 슈퍼컴퓨터든 임베디드든, **모든 컴퓨터는 동일한 5가지 구성 요소**를 갖는다:

```
                    ┌──────────────────────┐
                    │      Computer        │
   Input ──────►   │  ┌────────────────┐  │  ──────► Output
                    │  │   Processor    │  │
                    │  │ ┌──────┐┌────┐│  │
                    │  │ │Control││Data││  │
                    │  │ │      ││path││  │
                    │  │ └──────┘└────┘│  │
                    │  └───────┬────────┘  │
                    │          │           │
                    │  ┌───────▼────────┐  │
                    │  │    Memory      │  │
                    │  └────────────────┘  │
                    └──────────────────────┘
```

**Input/Output:**
- User-interface devices: 디스플레이(출력), 키보드·마우스(입력)
- Storage devices: 하드디스크, CD/DVD, 플래시, SSD
- Network adapters: 다른 컴퓨터와의 통신

**Processor (CPU):**
- **Datapath:** 실제 데이터 연산을 수행 (ALU, 레지스터 등)
- **Control:** 명령어를 해석하고 datapath에 적절한 제어 신호를 전달

**Memory:** 실행 중인 프로그램과 데이터를 저장.

#### 디스플레이: Through the Looking Glass

LCD 화면은 **픽셀(pixel)**의 집합으로 이미지를 표현한다. 메모리에 있는 **프레임 버퍼(frame buffer)**의 내용이 화면에 그대로 반영된다.

```
   Frame Buffer (메모리)              LCD 화면
  ┌─────────────┐              ┌─────────────┐
  │  (X0,Y0)=01 │   ------→   │  ■           │
  │  (X1,Y1)=10 │   ------→   │       ■      │
  └─────────────┘              └─────────────┘
```

#### 입력 장치: Optical Mouse

광학 마우스는 기계식 롤러볼 마우스를 대체했다. LED가 바닥면을 비추고, 내장 소형 저해상도 카메라가 표면 이미지를 촬영한 뒤, 내장 이미지 프로세서가 연속 프레임 간의 x, y 이동을 계산한다. 이것 자체가 하나의 소형 임베디드 컴퓨터이다.

#### 터치스크린: Touching the Glass

PostPC 시대를 상징하는 입력 장치. 두 가지 기술:

| 방식 | 원리 | 특징 |
|------|------|------|
| **Resistive** | 두 전도층의 물리적 접촉 | 저렴, 단일 터치 |
| **Capacitive** | 손가락의 정전용량 변화 감지 | 멀티터치 가능, 대부분의 스마트폰/태블릿 |

#### Opening the Box — 실제 하드웨어 내부

2008년 MacBook vs 2013년 MacBook Air 내부 비교:

**2008년:** Hard drive, Processor (with cover), Fan, Memory DIMMs (별도 슬롯), Battery (별도 공간), Motherboard, DVD drive

**2013년:** 배터리가 내부 공간 대부분 차지. 메모리는 메인보드에 직접 납땜(슬롯 없음). HDD 대신 플래시 스토리지. DVD 드라이브 제거.

PMD 시대의 설계 철학: 배터리 용량 극대화를 위해 모듈화를 포기하고 집적도를 높인다.

iPad 2 (A1395) 분해에서도 동일한 경향: Apple A5 프로세서, 플래시 스토리지, 대형 배터리가 하나의 보드에 집적.

#### Inside the Processor (CPU) 상세

CPU 내부의 세 가지 핵심 구성 요소:

- **Datapath:** 데이터에 대한 연산 수행 (ALU에서 덧셈, 뺄셈 등)
- **Control:** Datapath, 메모리 등의 동작 순서를 제어. 명령어 해석(decode)하고 제어 신호 생성
- **Cache Memory:** 빠른 데이터 접근을 위한 소형 고속 SRAM

세 시대의 프로세서 비교:

**AMD Barcelona (2007) — 4코어:**

```
┌──────────────────────────────────────┐
│  ┌─Core 1──┐  ┌─Core 2──┐           │
│  │128b FPU │  │128b FPU │           │
│  │L/S │L1D │  │L/S │L1D │           │
│  │Exec│L2  │  │Exec│L2  │           │
│  │F/D │L1I │  │F/D │L1I │           │
│  │    │512K│  │    │512K│           │
│  └────┘────┘  └────┘────┘           │
│          ┌──────────┐                │
│          │Northbridge│  ← 메모리 컨트롤러│
│          └──────────┘                │
│  ┌─Core 4──┐  ┌─Core 3──┐           │
│  │  ...    │  │  ...    │           │
│  └─────────┘  └─────────┘           │
│     ┌────────────────┐              │
│     │ 2MB Shared L3  │              │
│     └────────────────┘              │
└──────────────────────────────────────┘
  ↕ HyperTransport links    ↕ DDR PHY
```

각 코어마다 L1 Data Cache, L1 Instr Cache, 512KB L2 Cache. 4코어가 2MB Shared L3 Cache 공유. Northbridge가 메모리 컨트롤러 역할.

**Intel Arrow Lake (2024) — 이종 하이브리드:**

**Lion Cove P-core** (Performance, 8개) + **Skymont E-core cluster** (Efficiency, 4개 클러스터). 큰 작업은 P-core가, 백그라운드 작업은 E-core가 처리. L3 Cache는 Ring Agent를 통해 모든 코어가 공유, 각 블록 2×1.5MB씩 총 약 36MB.

**Apple A18 Pro (2025) — 모바일 SoC:**

CPU 코어, 5-core GPU, Neural Engine, 메모리 컨트롤러 등이 하나의 칩에 통합. PostPC 시대 "전력 효율 최우선" 설계 철학의 산물.

**핵심 통찰:** 2007년에는 CPU만 있는 단순 멀티코어였지만, 2024~2025년에는 이종 코어(big.LITTLE/P+E), GPU, NPU까지 하나의 칩에 집적하는 SoC 형태로 진화. 8가지 아이디어 중 "Parallelism"과 "Common Case Fast"의 구현.

### §1.4 (마무리) — Abstractions: The BIG Picture

**Abstraction(추상화)**은 복잡성을 다루는 가장 중요한 도구이다.

컴퓨터 구조에서 가장 중요한 세 가지 추상화 경계:

```
┌─────────────────────────────────────┐
│           Application               │
├─────────────────────────────────────┤
│        High-Level Language          │
├═══════════════ ABI ═════════════════┤  ← ISA + System SW interface
│  ┌──────────┐  ┌──────────────────┐ │
│  │    OS    │  │    Compiler      │ │
│  └──────────┘  └──────────────────┘ │
├═══════════════ ISA ═════════════════┤  ← HW/SW interface
│         Hardware                    │
│   (Implementation)                  │
└─────────────────────────────────────┘
```

**ISA (Instruction Set Architecture):** 하드웨어와 소프트웨어 사이의 인터페이스. "이 프로세서가 어떤 명령어를 이해하는가"를 정의. RISC-V, ARM, x86 등이 ISA.

**ABI (Application Binary Interface):** ISA에 시스템 소프트웨어의 인터페이스(시스템콜 규약, 레지스터 사용 규약 등)를 더한 것. 같은 ISA라도 OS가 다르면 ABI가 달라서 바이너리 호환이 안 될 수 있다.

**Implementation:** 인터페이스 아래의 실제 구현 세부사항. 같은 ISA(예: x86)라도 구현이 다를 수 있다 (Intel과 AMD가 같은 x86 ISA를 다르게 구현).

**왜 이 구분이 중요한가?** ISA가 동일하면 그 위의 소프트웨어는 하드웨어 구현이 바뀌어도 그대로 동작한다. 이것이 "소프트웨어 호환성"의 근간이며, Intel이 40년간 x86 ISA를 유지하면서도 내부 구현을 완전히 바꿀 수 있었던 이유이다.

### Understanding Performance

프로그램 성능에 영향을 미치는 4가지 요소:

| 요소 | 결정하는 것 |
|------|-----------|
| **Algorithm** | 실행되는 연산의 수 (operation count) |
| **Programming language, Compiler, Architecture** | 연산당 기계어 명령어 수 |
| **Processor, Memory system** | 명령어 실행 속도 |
| **I/O system (including OS)** | I/O 연산 실행 속도 |

이 4가지 요소는 서로 독립적이지 않고 연결되어 있다. 예를 들어 알고리즘이 연산 수를 줄이면 전체 실행 시간이 줄고, 컴파일러가 더 효율적인 기계어를 생성하면 같은 알고리즘이라도 더 빠르다.
