## Chapter 1: Computer Abstractions and Technology (60 slides)

### §1.1 컴퓨터 혁명

무어의 법칙: IC 트랜지스터 수가 약 2년마다 2배 증가하는 경험적 관찰. Intel 4004(1971, 2300개) → 72-core Xeon(2018, 수십억). 이 법칙 덕분에 자동차 컴퓨터, 스마트폰, 유전체 프로젝트, WWW, 검색엔진, ML/AI가 가능해졌다. 주의: 물리법칙이 아니며, 미세공정 한계로 둔화 → 병렬 처리가 중요해진 핵심 이유.

컴퓨터 분류: PC(비용/성능 트레이드오프), Server(높은 용량·성능·신뢰성), Supercomputer(최고 성능, 시장 비중 작음), Embedded(전력/성능/비용 제약 엄격, 수량 최다). PostPC 시대(2011~): PMD(배터리, 인터넷, 스마트폰/태블릿) + Cloud Computing(WSC, SaaS). 설계 핵심이 "절대 성능"에서 "전력 효율"로 전환.

IT 시가총액 추이: 2017 Apple 1위(754B) → 2026 NVIDIA 1위(4,328B). 반도체 기업(TSMC, Broadcom, SK Hynix, Micron, AMD) 상위권 대거 진입. AI/ML 수요 폭증 반영.

### §1.2 Eight Great Ideas

이 과목 전체를 관통하는 8가지 설계 원칙: (1) Design for Moore's Law — 완성 시점 기술에 맞춰 설계, (2) Use Abstraction — 복잡성을 계층적으로 숨김, (3) Make Common Case Fast — 자주 발생하는 경우를 빠르게(→Amdahl's Law), (4) Performance via Parallelism, (5) Performance via Pipelining — Ch4, (6) Performance via Prediction — Branch prediction, (7) Hierarchy of Memories — Ch5 캐시, (8) Dependability via Redundancy — RAID, ECC.

### §1.3 Below Your Program

소프트웨어 3계층: Application(HLL) → Systems SW(Compiler: HLL→기계어, OS: I/O·메모리·스케줄링) → Hardware(Processor, Memory, I/O). 프로그램 코드 수준: C(HLL) → [Compiler] → RISC-V Assembly → [Assembler] → Binary Machine Code. swap 예시에서 C의 3줄이 RISC-V 7개 명령어로 변환.

### §1.4 Under the Covers

컴퓨터 5대 구성: Input, Output, Processor(Datapath+Control), Memory, Cache. 모든 컴퓨터(Desktop~Embedded)가 동일한 구조. Datapath: 데이터 연산(ALU, 레지스터). Control: 명령어 해석, 제어 신호. Cache: CPU 내부 소형 고속 SRAM.

디스플레이: Frame Buffer의 비트값이 LCD 픽셀에 반영. 터치스크린: Resistive(물리적 접촉, 단일터치) vs Capacitive(정전용량 변화, 멀티터치).

프로세서 진화: AMD Barcelona 2007(동종 4코어, 2MB L3 공유) → Intel Arrow Lake 2024(이종 P-core 8개 + E-core 클러스터 4개, 36MB L3) → Apple A18 Pro 2025(CPU+GPU+NPU 단일 SoC). "Parallelism"과 "Common Case Fast"의 구현.

Abstraction — The BIG Picture: ISA(HW/SW 인터페이스, RISC-V/ARM/x86), ABI(ISA + 시스템콜/레지스터 규약), Implementation(마이크로아키텍처). ISA 동일하면 구현 바뀌어도 SW 호환. Intel이 40년간 x86 유지하면서 내부 구현을 바꿀 수 있었던 이유.

Volatile(DRAM, 전원off시 소실) vs Non-volatile(HDD, SSD, Flash, 보존). Networks: LAN(Ethernet), WAN(Internet), Wireless(Wi-Fi, Bluetooth, NFC).

### §1.5 Technologies

기술 발전: 진공관(1951) → 트랜지스터(1965) → IC(1975) → VLSI(1995) → ULSI(2005). 성능/비용 비율 62억배 향상. 반도체 기술: 실리콘에 재료 추가하여 도체/절연체/스위치 생성.

IC 제조: Silicon Ingot → Slicer → Blank Wafers → 20-40 processing steps → Patterned Wafers → Wafer Tester → Dicer → Tested Dies → Package → Part Tester → Ship. Yield = 정상 다이/전체 다이. Cost/die = Cost_wafer/(Dies/wafer × Yield). 다이 면적↑ → 수율↓ → 비용 비선형↑.

### §1.6 Performance

Response Time(작업 완료 시간) vs Throughput(단위시간 총 작업량). Performance = 1/Execution Time. "X가 Y보다 n배 빠르다" = Time_Y/Time_X.

Elapsed Time(전체, I/O 포함) vs CPU Time(순수 연산). CPU Clocking: Clock Period(e.g. 250ps) = 1/Clock Rate(e.g. 4GHz).

★ CPU Time = Clock Cycles × T_c = Clock Cycles / Clock Rate. 성능 향상: 사이클 수↓ 또는 클럭↑ (트레이드오프).

★ CPU Time = IC × CPI × T_c. IC: 프로그램/ISA/컴파일러 결정. CPI: CPU HW 결정. Weighted CPI = Σ(CPI_i × IC_i/IC_total).

CPU Time 예시: A(2GHz,10s), B 목표(6s,1.2×cycles) → Cycles_A=20G, Cycles_B=24G → Rate_B=4GHz.

CPI 예시: A(T_c=250ps,CPI=2.0), B(T_c=500ps,CPI=1.2), 같은 ISA → Time_A=500·IC, Time_B=600·IC → A가 1.2배 빠름.

CPI mix 예시: Class A(CPI=1),B(2),C(3). Seq1(2,1,2): 10cycles/5IC=CPI2.0. Seq2(4,1,1): 9cycles/6IC=CPI1.5. IC 적다고 항상 빠른 것 아님.

Performance Summary (BIG Picture): CPU Time = IC × CPI × T_c. Algorithm→IC,CPI. Language→IC,CPI. Compiler→IC,CPI. ISA→IC,CPI,T_c.

### §1.7 The Power Wall

Power = C × V² × f (CMOS 동적 전력). 지난 수십년: Clock×1000, V 5V→1V, Power×30으로 관리. 이제 전압 더 못 낮추고 발열 더 못 제거 → Power Wall → 클럭 향상 한계 → 멀티코어 전환.

전력 절감 예시: 85% C, 85% V, 85% f → P_new/P_old = 0.85⁴ ≈ 0.52 (48% 절감).

### §1.8 Switch to Multiprocessors

단일 프로세서 성능 향상률: 25%/yr(~1986) → 52%/yr(1986-2003, ILP 전성기) → 23%(2003-2011) → 12%(2011-2015) → 3.5%/yr(2015~). 멀티코어: 명시적 병렬 프로그래밍 필요(ILP는 HW가 자동). 어려움: 성능 프로그래밍, 부하 균형, 통신/동기화.

### §1.9 SPEC Benchmarks

SPEC CPU2006/2017: 실제 프로그램 실행시간 측정, 참조 머신 대비 정규화, 기하 평균으로 요약. SPECpower: 부하 수준별 ssj_ops/Watt. Energy proportionality 문제(10%부하에서도 47% 전력).

### §1.10 Fallacies and Pitfalls

★ Amdahl's Law: T_improved = T_affected/n + T_unaffected. 예시: 100s 중 80s 개선, 5× 전체? → 80/n+20=20 → 불가능! → "Make the common case fast"의 수학적 근거.

Fallacy: Low Power at Idle (i7: 100%→258W, 10%→121W). Pitfall: MIPS = Clock_rate/(CPI×10⁶) — ISA/복잡도 차이 무시→신뢰 불가.

## Chapter 2: Instructions — Language of the Computer (95 slides)

### §2.1 Introduction

Instruction Set: 컴퓨터가 이해하는 명령어의 전체 목록. 서로 다른 ISA지만 공통 측면 많음(덧셈, 메모리접근, 분기). RISC 철학: 단순한 명령어 → 구현 간결. RISC-V: UC Berkeley 개발 오픈 ISA, RISC-V Foundation 관리, 임베디드 코어 시장 대비중. 참고: MIPS(Stanford), ARM(Acorn).

### §2.2 Operations

산술 명령어: 항상 3-오퍼랜드(2 source + 1 dest). add a,b,c // a=b+c. Design Principle 1: "Simplicity favors regularity" — 규칙적 형식→HW 간결→고성능 저비용.

예시: f=(g+h)-(i+j) → add x5,x20,x21 / add x6,x22,x23 / sub x19,x5,x6.

### §2.3 Operands

레지스터: 32×64-bit(x0~x31). DP2: "Smaller is faster". x0=상수0, x1=ra, x2=sp, x5-7/x28-31=temp(비보존), x8-9/x18-27=saved(보존), x10-17=인자/결과.

메모리 오퍼랜드: 배열/구조체는 메모리. ld(load)/sd(store)로 레지스터↔메모리. Byte addressed, Little Endian, 정렬 제약 없음. doubleword offset = index×8. 예시: A[12]=h+A[8] → ld x9,64(x22) / add x9,x21,x9 / sd x9,96(x22).

Registers vs Memory: 레지스터가 더 빠름. 컴파일러는 변수를 최대한 레지스터에 배치, 부족 시 spill.

Immediate: addi x22,x22,4. "Make the common case fast" — 작은 상수가 흔하므로 별도 load 불필요.

### §2.4 Signed and Unsigned Numbers

Unsigned n-bit: 0~2^n-1. 2's complement: -2^(n-1)~2^(n-1)-1. Bit63=부호비트. 0=0...0, -1=1...1. 부정: 비트반전+1. Sign extension: 부호비트를 왼쪽으로 복제. lb(sign extend), lbu(zero extend).

### §2.5 Representing Instructions

RISC-V 명령어: 32-bit 고정 길이. Hexadecimal: 4bits=1hex digit.

R-type(레지스터 산술): funct7|rs2|rs1|funct3|rd|opcode (7|5|5|3|5|7 bits). 예: add x9,x20,x21 → 0000000|10101|10100|000|01001|0110011 = 0x015A04B3.

I-type(즉시값/Load): imm[11:0]|rs1|funct3|rd|opcode (12|5|3|5|7). S-type(Store): imm[11:5]|rs2|rs1|funct3|imm[4:0]|opcode. SB-type(분기): PC-relative, Target=PC+imm×2. U-type(lui): imm[31:12]|rd|opcode. UJ-type(jal): 20-bit imm.

DP3: "Good design demands good compromises" — 여러 포맷이지만 rs1/rs2 위치 통일.

Stored Program: 명령어도 이진수로 메모리 저장. 프로그램이 프로그램 조작 가능(컴파일러, 링커).

### §2.6 Logical Operations

sll/slli(좌시프트,×2^i), srl/srli(우논리시프트,÷2^i unsigned), sra/srai(우산술시프트, 부호비트채움). and/andi(비트마스킹), or/ori(비트포함), xor/xori(비트차이, xor all 1s=NOT). RISC-V는 별도 NOT 없이 xor로 구현.

### §2.7 Making Decisions

beq(==), bne(!=), blt(<signed), bge(>=signed), bltu/bgeu(unsigned). 무조건분기: beq x0,x0,L 또는 jal x0,L.

if-else: if(i==j) f=g+h; else f=g-h; → bne x22,x23,Else / add x19,x20,x21 / beq x0,x0,Exit / Else: sub x19,x20,x21 / Exit:

while: while(save[i]==k) i+=1; → Loop: slli x10,x22,3 / add x10,x10,x25 / ld x9,0(x10) / bne x9,x24,Exit / addi x22,x22,1 / beq x0,x0,Loop / Exit:

Signed vs Unsigned: 같은 비트 1111...1111이 signed에서 -1, unsigned에서 4,294,967,295. Basic Block: 중간 분기 없는 명령어 시퀀스, 최적화·가속 기본단위.

### §2.8 Procedures

6단계: 인자→x10-17, 제어이전, 저장공간확보, 연산, 결과→x10-11, 복귀→x1. jal x1,Label(call), jalr x0,0(x1)(return).

Leaf procedure(leaf_example): saved regs(x5,x6,x20) 스택 저장→연산→복원→리턴. 스택: sp-=24, sd 3개, 연산, ld 3개, sp+=24, jalr.

Non-leaf(factorial): 반드시 x1(ra)과 인자를 스택에 저장. fact: sp-=16, sd x1/x10, n<1이면 return 1, 아니면 addi x10,-1 / jal x1,fact(재귀) / ld x10/x1 / mul x10,x10,x6 / jalr.

Register Usage: temp(x5-7,x28-31) callee 비보존, saved(x8-9,x18-27) callee 보존.

Memory Layout: Stack(↓grows down, SP→top) | Dynamic data/Heap(↑grows up) | Static data(gp→) | Text(PC→) | Reserved(0).

### §2.9 Communicating with People

ASCII(128), Latin-1(256), Unicode(32-bit, UTF-8/16). lb/lh/lw(sign extend), lbu/lhu/lwu(zero extend), sb/sh/sw. strcpy 예시: lbu로 바이트 읽기, sb로 쓰기, null 체크.

### §2.10 Wide Immediates and Addresses

32-bit 상수: lui rd,imm20(rd[31:12]=imm, sign extend, [11:0]=0) + addi. Branch addressing: SB-format, PC+imm×2. Jump: UJ-format, 20-bit. 범위 초과 시 어셈블러가 bne+jal로 변환. 4가지 어드레싱: Immediate, Register, Base(displacement), PC-relative.

### §2.11 Synchronization

멀티프로세서 공유메모리 data race 방지. lr.d rd,(rs1): Load Reserved. sc.d rd,(rs1),rs2: Store Conditional(성공:rd=0, 실패:rd≠0). atomic swap: lr.d/sc.d/bne 루프. lock: lr.d로 읽고 0인지 확인 후 sc.d로 1 설정.

### §2.12 Translating and Starting

C→[Compiler]→Assembly→[Assembler]→Object→+Library→[Linker]→Executable→[Loader]→Memory. Object module: Header, Text, Static data, Relocation, Symbol table, Debug info. Linking: 세그먼트 병합, 라벨 해결, 위치의존 참조 패칭. Loading 6단계. Dynamic Linking: 호출 시점 로드, Lazy Linkage.

### §2.13 C Sort Example

swap(leaf): slli x6,x11,3 / add x6,x10,x6 / ld x5,0(x6) / ld x7,8(x6) / sd x7,0(x6) / sd x5,8(x6) / jalr. sort(non-leaf): 이중 for, saved regs/ra 스택 저장, swap 호출.

### §2.14 Arrays vs Pointers

배열: 매번 index×size+base. 포인터: 주소 직접 증가. 컴파일러 induction variable elimination으로 동일 최적화→배열 권장.

### §2.16-2.17 Real Stuff

RISC-V vs MIPS: 거의 동일 철학. 차이: 조건분기 방식(RISC-V: blt/bge 직접, MIPS: slt+beq 2단계). x86 진화: 8080→8086→80386→Pentium→AMD64→SSE/AVX. 가변길이, 2-오퍼랜드, 내부적으로 RISC 마이크로연산 변환 실행. "기술적 우아함 ≠ 시장 성공".

### §2.18 RISC-V Extensions

I(Base integer), M(Multiply/Divide), A(Atomic), F(Single FP), D(Double FP), C(Compressed 16-bit).

### §2.19-2.20 Fallacies, Pitfalls, Concluding

Fallacy: 강력한 명령어=높은 성능(✗). Fallacy: 어셈블리가 항상 더 빠름(✗). Pitfall: 순차 word가 순차 주소(✗, +8!). Pitfall: 지역변수 포인터를 리턴 후 사용(✗, 스택 pop 무효).

3 Design Principles: (1) Simplicity favors regularity, (2) Smaller is faster, (3) Good design demands good compromises. + Make the common case fast.

SPEC2006 명령어 빈도: Int(Arith16%, DataXfer35%, Logic12%, Branch34%, Jump2%), FP(Arith48%, DataXfer36%, Logic4%, Branch8%, Jump0%).

## Chapter 3: Arithmetic for Computers (52 slides)

### §3.1-3.2 정수 덧셈/뺄셈과 오버플로우

이진 덧셈은 LSB부터 캐리를 전달하며 수행. 오버플로우 조건: 양수+양수→결과 부호 1(음수), 음수+음수→결과 부호 0(양수)이면 overflow. 부호가 다른 피연산자끼리는 overflow 불가능.

뺄셈은 2's complement 부정+덧셈: 7-6 = 7+(-6). 오버플로우 조건도 유사하게 적용.

SIMD: 64-bit 가산기에 캐리체인 분할하여 8×8-bit, 4×16-bit, 2×32-bit 벡터 동시 처리. Saturating: overflow 시 최대/최소값으로 고정(clamp), 오디오 클리핑/비디오 채도에 사용.

### §3.3 곱셈

이진 long multiplication: 1000×1001 = 1001000. n-bit×n-bit → 2n-bit 결과. 곱셈 HW: Multiplicand(128b, shift left), Multiplier(64b, shift right), Product(128b, initially 0), 128-bit ALU. 32회 반복: multiplier bit0 검사→1이면 product+=multiplicand→multiplicand shift left→multiplier shift right.

Optimized: add/shift 병렬 수행, 1 cycle/partial-product. Faster: 다수 adder 사용, 파이프라이닝 가능. RISC-V: mul(하위 64b), mulh/mulhu/mulhsu(상위 64b). mulh로 overflow 검출.

### §3.4 나눗셈

Long division: 1001010÷1000 = 몫1001, 나머지10. n-bit → n-bit 몫 + n-bit 나머지. Restoring division: 빼기 후 음수이면 divisor 다시 더함. 곱셈보다 병렬화 어려움(뺄셈이 나머지 부호에 conditional). SRT division: 한 단계에 여러 몫 비트 생성.

RISC-V: div/rem(signed), divu/remu(unsigned). 0 나누기/오버플로우 시 에러 없이 정의된 결과 반환(common case fast).

### §3.5 부동소수점 (IEEE 754)

필요성: 정수로는 소수, 매우 큰/작은 수 표현 불가. 과학적 표기법: ±1.xxx₂ × 2^yyy (normalized).

IEEE 754 포맷: [S|Exponent|Fraction]. Single: 1+8+23=32bit, Bias=127. Double: 1+11+52=64bit, Bias=1023. 값 = (-1)^S × (1+Fraction) × 2^(Exponent-Bias). Hidden bit: 정규화 시 정수부 1은 저장 안 함.

범위: Single ±1.2×10⁻³⁸~±3.4×10³⁸ (~6자리). Double ±2.2×10⁻³⁰⁸~±1.8×10³⁰⁸ (~16자리).

변환 예시: -0.75 = -1.1₂×2⁻¹ → S=1, Frac=100...0, Exp=126(01111110). 역변환: 11000000101000...0 → S=1, Exp=129, Frac=01 → (-1)×1.25×2²=-5.0.

특수값: Exp=0,Frac=0→±0. Exp=0,Frac≠0→Denormal(hidden bit=0, gradual underflow). Exp=all1,Frac=0→±Infinity. Exp=all1,Frac≠0→NaN(0/0 등).

FP 덧셈 4단계: (1)지수 정렬(작은 쪽 shift right) (2)가수 덧셈 (3)정규화+over/underflow 검사 (4)반올림+재정규화. 예시: 9.999×10¹+0.016×10¹=10.015×10¹→1.002×10².

FP 곱셈 5단계: (1)지수 덧셈(biased이면 bias 빼기) (2)가수 곱셈 (3)정규화 (4)반올림 (5)부호 결정. 핵심 차이: 덧셈은 지수정렬+가수덧셈, 곱셈은 지수덧셈+가수곱셈.

FP Adder HW: 정수 가산기보다 훨씬 복잡. 4단계 파이프라인(exponent비교→shift정렬→BigALU덧셈→정규화/반올림). 여러 사이클 필요, 파이프라이닝 가능.

RISC-V FP: 별도 레지스터 f0~f31. flw/fld(load), fsw/fsd(store). fadd.s/d, fsub.s/d, fmul.s/d, fdiv.s/d, fsqrt.s/d. 비교: feq/flt/fle → 정수 레지스터에 0/1 → beq/bne로 분기.

예시: f2c(float fahr) → flw const5/9/32, fdiv.s(5/9), fsub.s(fahr-32), fmul.s 결과.

### §3.6-3.7 SIMD와 x86 FP

Subword parallelism: 128-bit adder로 16×8b/8×16b/4×32b 동시 처리. SSE2(128b), AVX(256b). 행렬 곱셈에서 AVX intrinsics로 4×double 병렬→큰 성능 향상.

x86 FP: 8087의 8×80-bit 스택→코드 생성 어려움→SSE2/AVX가 대체.

### §3.8-3.9 Accurate Arithmetic, Fallacies

Guard/Round/Sticky bits: 중간 계산 정밀도 유지, 정확한 반올림. Pitfall: sra(산술 우시프트)는 음수에서 -∞ 방향 반올림, 프로그래밍 언어 나눗셈(0 방향)과 다름. FP associativity 불성립: (A+B)+C ≠ A+(B+C) 가능 → 병렬 프로그램 주의. Pentium FDIV 버그: 시장은 정확성을 기대.

### §3.10 결론

비트 자체에 의미 없음 — 명령어에 따라 해석. 유한 범위·정밀도 → overflow/underflow/rounding 고려 필수. RISC-V Core 54 명령어로 SPECINT 100%, SPECFP 97% 커버.
